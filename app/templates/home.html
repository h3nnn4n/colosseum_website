Colosseum is a fully automated platform where agents compete against each other on multiple games. Games run 24/7, non stop. Matches are organized in tournaments, having different formats, and tournaments are grouped in seasons.

Agents can be implemented using whatever technology and programming language that the author wants. It can be a simple script applying an static strategy, it could be using a deep learning model, reinforcement learning, etc. It is even possible to have the logic hosted on a webservice and run it remotly, to get around resource limitations. The only constraints are the rules of the game, which usually have a limit on how fast the agent has to respond.

Sample Agents

We have a collection of sample agents for all the games available on: colosseum_agents. They are MIT licensed. Feel free to modify them and use it as an starting point.

Getting Started

To get started please it is necessary to have an account. If you don't have one you can create one here.

Next, you need an agent. The agent is the robot program that will play the games, based on some strategy that it was programmed to follow. We have some basic agents available in our repository colosseum_agents to use as a reference.

Alternativelly, you can start from scratch using one of our SDK's from our repository: colosseum_sdk. It is also possible to interface directly with Colosseum. The communication protocol is quite simple and can be implemented in any language. Each game has it's own set of commands, so please refer to the games wiki page and pick one for a complete protocol documentation. An alternative up-to-date list is also available here.
